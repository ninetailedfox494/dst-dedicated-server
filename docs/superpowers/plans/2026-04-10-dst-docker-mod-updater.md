# DST Docker Mod Updater Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Docker-native workflow to reset and update DST mods from Steam Workshop on demand.

**Architecture:** Add a one-shot `mod-updater` Compose service that runs a dedicated update script. The script reads `env/mods.txt`, clears old mod/cache/config, downloads newest workshop mods with `validate`, regenerates `dedicated_server_mods_setup.lua` and shard `modoverrides.lua`, then exits with a summary. A host wrapper script orchestrates stop → update → start.

**Tech Stack:** Bash, Docker Compose, SteamCMD, DST dedicated server files, Markdown

#!/usr/bin/env bash
set -euo pipefail

DST_CLUSTER_NAME="${DST_CLUSTER_NAME:-MyDediServer}"
DST_CLUSTER_ROOT="${DST_CLUSTER_ROOT:-/home/dst/.klei/DoNotStarveTogether}"
DST_CLUSTER_DIR="${DST_CLUSTER_DIR:-${DST_CLUSTER_ROOT}/${DST_CLUSTER_NAME}}"
DST_INSTALL_DIR="${DST_INSTALL_DIR:-/home/dst/dst_server}"
MODS_FILE="${MODS_FILE:-/workspace/env/mods.txt}"

mkdir -p "${DST_INSTALL_DIR}/mods" "${DST_CLUSTER_DIR}/Master" "${DST_CLUSTER_DIR}/Caves"

mapfile -t MOD_IDS < <(grep -Ev '^\s*($|#)' "${MODS_FILE}" | tr -d '[:space:]')
if [[ ${#MOD_IDS[@]} -eq 0 ]]; then
  echo "ERROR: no mod IDs found in ${MODS_FILE}" >&2
  exit 1
fi

{
  echo "-- Auto-generated by reset_and_install_mods_docker.sh"
  for id in "${MOD_IDS[@]}"; do
    echo "ServerModSetup(\"${id}\")"
  done
} > "${DST_INSTALL_DIR}/mods/dedicated_server_mods_setup.lua"

{
  echo "return {"
  for id in "${MOD_IDS[@]}"; do
    echo "  [\"workshop-${id}\"] = { enabled = true, configuration_options = {} },"
  done
  echo "}"
} > "${DST_CLUSTER_DIR}/Master/modoverrides.lua"

cp "${DST_CLUSTER_DIR}/Master/modoverrides.lua" "${DST_CLUSTER_DIR}/Caves/modoverrides.lua"

if [[ "${DST_TEST_MODE:-0}" == "1" ]]; then
  exit 0
fi
```

# Add service in docker-compose.yml
  mod-updater:
    image: dst-dedicated:latest
    env_file:
      - ./env/.env
    environment:
      DST_SHARD_NAME: Master
      MODS_FILE: /workspace/env/mods.txt
    command: ["/workspace/scripts/reset_and_install_mods_docker.sh"]
    volumes:
      - ./:/workspace
      - ./data/cluster:/home/dst/.klei/DoNotStarveTogether/${DST_CLUSTER_NAME}
      - ./data/master:/home/dst/.klei/DoNotStarveTogether/${DST_CLUSTER_NAME}/Master
      - ./data/caves:/home/dst/.klei/DoNotStarveTogether/${DST_CLUSTER_NAME}/Caves
    profiles: ["tools"]
```

```bash
#!/usr/bin/env bash
# scripts/update_mods.sh
set -euo pipefail

echo "Stopping shards..."
docker compose stop dst-master dst-caves
echo "Running mod updater..."
docker compose --profile tools run --rm mod-updater
echo "Starting shards..."
docker compose up -d dst-master dst-caves
echo "Done."
```
